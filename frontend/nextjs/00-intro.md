## 1. What is Next.js?

Next.js is a **framework built on top of React**. React alone only handles client-side rendering (in the browser). Next.js adds everything needed to build a real production-ready application:

- Server-side rendering (SSR, SSG, ISR)
- Automatic file-based routing
- Built-in backend (Route Handlers / API)
- Automatic optimizations (images, fonts, caching)

## 2. Why Next.js (vs React alone)?

| React (Vite, CRA...) | Next.js |
|---|---|
| Client-Side Rendering only | SSR, SSG, ISR, CSR — your choice |
| Poor SEO (empty HTML at first) | Optimized SEO (pre-rendered HTML) |
| Manual routing (react-router) | Automatic file-based routing |
| No backend | Built-in backend (`app/api/`) |
| API keys exposed on the client | API keys protected on the server |

## 3. Required tools — and their role

### 1. Node.js — the engine
This is what lets JavaScript run **outside the browser** (on your computer/server). Without Node, JS only runs inside Chrome/Firefox. Next.js, like any modern tool, needs Node to work on the server side.

### 2. npm — Node Package Manager
The **package/library** manager. When you run `npm install react`, it downloads React and all its dependencies, and puts them into the `node_modules/` folder. It's basically an app store for JS code.

- `npm install` → installs a project's dependencies
- `npm run dev` → runs a script defined in `package.json` (e.g. starting the dev server)

### 3. npx — Node Package eXecute
Used to **run** a package **without installing it permanently**. That's what you use for `npx create-next-app`: it temporarily downloads the tool, runs it once to create your project, then doesn't keep it installed globally on your machine. Avoids cluttering your system with tools you only use once.

### 4. nvm — Node Version Manager
Manages **versions** of Node itself. Since Node evolves fast (new versions every 6 months), different projects may require different versions. nvm lets you keep Node 18 for an old project and Node 22 for Next.js 16, and switch between them with a single command (`nvm use 22`).

**Full chain:** `nvm` picks the version of → `Node` (the engine that runs the JS) → with `npm` installing the libs → and `npx` running one-off tools (like `create-next-app`).

### Issue encountered: `npx` not found
On Ubuntu, `npx` wasn't installed (neither was `npm`). Recommended fix: install **nvm**, then run `nvm install --lts` to get an up-to-date Node/npm/npx in a single command (avoids outdated versions from the `apt` repo).

## 4. Project installation

```bash
npx create-next-app@latest ./
```

Result:
- Installed version: `create-next-app@16.2.10`
- Choice: **recommended defaults** (Yes, use recommended defaults)
- Generated template: **app-tw** → App Router + Tailwind CSS
- Installed dependencies: `next`, `react`, `react-dom`
- Dev dependencies: `@tailwindcss/postcss`, `@types/node`, `@types/react`, `@types/react-dom`, `eslint`, `eslint-config-next`, `tailwindcss`, `typescript`

## 5. Generated folder structure

```
nextjs_demo/
├── .idea/                  ← WebStorm config (not related to Next.js)
├── .next/                  ← auto-generated build output, don't touch
├── app/                    ← App Router — the core of the project
│   ├── favicon.ico
│   ├── globals.css         ← global styles (Tailwind imported here)
│   ├── layout.tsx          ← root layout, required
│   └── page.tsx            ← home page (route "/")
├── node_modules/           ← all installed dependencies
├── public/                 ← static files (images, etc.)
├── .gitignore
├── eslint.config.mjs       ← ESLint config
├── next.config.ts          ← Next.js config
├── next-env.d.ts           ← Next.js types (auto-generated, don't edit)
├── package.json            ← project dependencies + scripts
├── package-lock.json
├── postcss.config.mjs      ← PostCSS config (used by Tailwind)
├── README.md
└── tsconfig.json           ← TypeScript config
```

### Content of `app/layout.tsx` (default generated)

```tsx
import type { Metadata } from "next";
import { Geist, Geist_Mono } from "next/font/google";
import "./globals.css";

const geistSans = Geist({
  variable: "--font-geist-sans",
  subsets: ["latin"],
});

const geistMono = Geist_Mono({
  variable: "--font-geist-mono",
  subsets: ["latin"],
});

export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  // ...
}
```

Key takeaways:
- `Geist` and `Geist_Mono` = Google fonts automatically optimized via `next/font/google` (zero layout shift)
- `metadata` = exported object for SEO (title, description) — automatically picked up by Next.js in the `<head>`
- `RootLayout` = the root layout, wraps every page, must contain `<html>` and `<body>`

## 6. "Snippets" extension for quickly generating React components

Shortcut `rafce` (+ Tab) automatically generates a component skeleton:

```tsx
import React from 'react'

const hello = () => {
  return (
    <div>hello</div>
  )
}

export default hello
```

- **VS Code** → extension "ES7+ React/Redux/React-Native Snippets" (author: dsznajder)
- **WebStorm** → extension "Modern React Snippets"

`Ctrl + Alt + S` → Settings/Preferences → **Plugins** → **Marketplace** tab → search for the plugin name.
